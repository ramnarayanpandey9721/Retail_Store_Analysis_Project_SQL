Retail Store SQL Analysis Project
Overview

This SQL-based retail analytics project focused on analyzing customer behavior, sales performance, and product trends using Point of Sale (POS) data. The project answers real-world business questions through optimized SQL queries and structured data analysis.

Business Problem

A retail company wants to understand:

Customer purchasing patterns

Product and category performance

Revenue, returns, and store-type effectiveness

The goal is to convert raw transactional data into actionable business insights.

Dataset

The analysis is performed on three relational tables:

Customer – demographic details (age, gender, city)

Transactions – purchase and return transactions

Product Category – product category and sub-category mapping

Key Analysis & Insights

Identified most frequently used sales channels and top-performing store types

Analyzed customer demographics by gender and city

Calculated net revenue by product category and sub-category

Evaluated sales vs returns percentage and return-heavy categories

Identified high-value and repeat customers

Performed time-based revenue analysis using transaction date ranges

SQL Skills Demonstrated

Complex JOIN operations

Aggregations (SUM, COUNT, AVG)

Subqueries and filtering logic

Date transformation and time-based analysis

Business-driven query optimization
